Active N.I.N.A. Instances

Selects which of the three N.I.N.A. instance slots the device polls and shows as pages. A disabled instance is not contacted and its page is skipped, but the slot keeps its fixed position so the other instances do not shift.

Each enabled instance gets its own dashboard page and a configuration card below. The device updates the page you are viewing in full and checks the other instances less often in the background.

Example: enable N.I.N.A. 1 and N.I.N.A. 2 for a two-instance setup and leave N.I.N.A. 3 off; only two node cards and two instance pages appear.

Polling & Connectivity
Polling and Connectivity

Sets how often the device contacts the NINA API and how patient it is before declaring an instance offline. Data update rate is the base polling interval for the active page; faster values refresh the display sooner but generate more API traffic. Graph update rate controls how often the RMS and HFR history graphs add a point.

Connection timeout is how long a request waits for a response before that instance is marked offline. Idle poll interval is the slower, occasional check used to re-test an offline instance while you are viewing a non-NINA page, which avoids contacting an instance that is not running too often.

Example: set Data update rate to 2 seconds for a responsive display, raise Connection timeout to 10 seconds on a slow or distant network, and leave Idle poll interval at 30 seconds.

Update Rates
Base polling interval for the active page, from 1 to 10 seconds; lower is more responsive but adds API traffic.
How often the RMS and HFR history graphs record a new data point, from 2 to 30 seconds.
